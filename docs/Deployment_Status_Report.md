# 🏥 CareLink 배포 및 로그인 이슈 해결 최종 리포트

**최종 업데이트**: 2026-03-16
**상태**: ✅ 해결 완료 (로그인 및 데이터 연동 정상 작동 확인)

---

## 1. 개요
본 리포트는 CareLink 프로젝트를 Netlify와 Supabase 환경에 배포하는 과정에서 발생한 주요 장애 요인들과 이를 해결한 과정을 상세히 기록합니다. 특히, 데이터베이스 연결부터 테스트 유저 로그인 실패 이슈를 해결한 최종 솔루션을 포함합니다.

---

## 2. 주요 장애 해결 과정

### 🛠️ 데이터베이스 연결 (Supabase)
- **이슈**: 국내 일반 인터넷(IPv4) 환경에서 Supabase의 기본 접속 주소(IPv6)를 찾지 못하는 `ENOTFOUND` 에러 발생.
- **해결**: Supabase의 **Transaction Pooler (포트 6543)** 전용 주소를 사용하여 IPv4 환경에서도 안정적으로 커넥션을 유지하도록 설정하였습니다.
- **도구**: `backend/src/init_db.js`를 통해 테이블 스키마를 자동 구축하고 `seed.js`로 테스트 데이터를 주입했습니다.

### 🔍 로그인 실패 이슈: "가입되지 않은 이메일 (입력길이: 0)"
- **원인 분석**: 
    1. **데이터 유실**: 서버리스 환경에서 클라이언트가 보낸 JSON Body가 서버에 도달하기 전 증발하거나 조각나는 현상 발생.
    2. **대소문자/공백 이슈**: 입력 시 포함된 미세한 공백이나 대소문자 불일치로 인한 DB 조회 실패.
- **해결 조치**: 
    - 프론트엔드 전송 시 `trim()` 및 `toLowerCase()`를 적용하여 데이터를 정제함.
    - Axios 전역 설정에 `Content-Type: application/json`을 강제하여 데이터 유실 방지.

### 🕵️ 결정적 단서: 조각난 바디 (Indexed Body)
- **이슈**: 디버깅 로그 확인 결과, 데이터가 `{"0": "{", "1": "\"", ...}` 처럼 **인덱스 키를 가진 객체**로 쪼개져서 들어오는 특이 현상 발견.
- **솔루션**: 백엔드 `app.js`에 **조각난 바디 복구 미들웨어**를 추가했습니다. 숫자로 된 키를 가진 객체가 들어오면 이를 다시 합쳐서 정상적인 JSON으로 복구하는 Fallback 로직입니다.

---

## 3. 최종 확인 결과 (로컬 및 재배포 대비)

- **로컬 검증 완료**: 조각난 데이터를 강제로 주입하는 테스트 코드를 통해 서버가 이를 완벽히 복구하고 로그인을 처리하는 것을 확인했습니다.
- **데이터 실시간 연동**: 로그인 후 MyPage 진입 시 `/reports/years`와 `/reports/health` API가 정상적으로 호출되어 실제 건강 점수와 차트가 화면에 출력됨을 확인했습니다. (유저: 박지연 님 데이터 확인됨)

---

## 4. 새로운 계정 배포 시 주의사항

1. **환경 변수 선입력**: Netlify 프로젝트 생성 시 `VITE_API_URL`, `DATABASE_URL`, `JWT_SECRET`, `GEMINI_API_KEY`를 먼저 입력한 후 빌드를 시작하십시오.
2. **비밀번호 특수문자**: `DATABASE_URL`에 특수문자(`@`, `:`)가 포함된 경우 연결 오류가 발생할 수 있으므로 가급적 영문/숫자 조합의 비밀번호를 사용하시기 바랍니다.
3. **SPA 라우팅**: 새로고침 시 404 에러를 방지하기 위해 `public/_redirects` 파일이 포함되었습니다.
