# 🚨 현재 배포 상태 및 미해결 이슈 리포트

**최종 업데이트**: 2026-03-16
**상태**: 🟡 부분 완료 (인프라 구축 완료, 기능 이슈 분석 중)

---

## 1. 개요
현재 CareLink 프로젝트는 Netlify(프론트/백)와 Supabase(DB)를 연동하여 배포를 완료했습니다. 하지만 **실제 운영 환경에서의 로그인 기능**이 특정 환경 이슈로 인해 현재 미해결 상태입니다. 팀원들은 아래 내용을 숙지하여 개발 및 디버깅에 참고해주시기 바랍니다.

---

## 2. 미해결 이슈: 로그인 유효성 검사 실패

### 🔍 증상
- 로그인 시도 시 서비스에서 **"가입되지 않은 이메일입니다. (입력길이: 0)"** 에러 발생.
- 브라우저 콘솔에는 `400 Bad Request`가 기록됨.

### 🕵️ 분석 결과 (범인: Serverless Body 유실)
- **원인**: Netlify Functions(서버리스 환경)에서 클라이언트가 보낸 JSON 데이터를 Express 서버가 제대로 수신하지 못하고 있습니다. 
- **증거**: 서버 로그 확인 결과, `req.body`가 빈 상태(`{}`)로 들어와 이메일 입력값이 서버에 도달하지 못해 발생하는 현상입니다.
- **상태**: 현재 이를 해결하기 위한 강제 바디 파싱(Raw Body Parser) 코드를 적용했으나, **Netlify 빌드 한도 초과**로 인해 최신 코드가 아직 반영되지 않았을 가능성이 높습니다.

---

## 3. 기능 점검 상황 (정상 작동 확인됨)

이슈에도 불구하고 DB 연동 자체는 완벽함을 확인했습니다.
- **DB 접속**: 서버 -> Supabase 접속 성공 (`debug-db` 경로 확인 완료)
- **데이터 존재**: `test@test.com` 계정이 DB에 정상 등록되어 있음을 시뮬레이션으로 확인.
- **로컬 환경**: 로컬에서 수파베이스 DB를 바라보고 실행 시 모든 기능이 **정상 작동**함.

---

## 4. 에러 핸들링 가이드 (팀원 참고용)

향후 디버깅을 위해 다음 도구들을 마련해두었습니다.

1. **진단용 경로**: `https://[배포주소]/.netlify/functions/api/debug-db`
   - 현재 서버가 연결된 DB 정보와 유저 데이터 존재 여부를 즉시 보여줍니다.
2. **세밀한 에러 메시지**: 
   - 로그인 실패 시 단순히 "실패"가 아니라, 서버가 받은 데이터의 길이와 구체적인 이유를 메시지에 포함하도록 설계했습니다.
3. **자동화 스크립트**:
   - `node src/init_db.js`: 테이블 자동 생성
   - `node src/seed.js`: 테스트 데이터 자동 주입

## 5. 향후 작업 계획
1. **Netlify 환경 점검**: 빌드 한도 리셋 또는 유료 플랜 검토, 혹은 Vercel로의 마이그레이션 검토.
2. **Body-Parser 보강**: 서버리스 환경에 특화된 `event.body` 직접 접근 로직 검토.
3. **팀원 협력**: 로컬 환경에서 `DATABASE_URL`을 Supabase로 맞춰서 기능 테스트를 진행해주시고, 서버리스 환경에서의 이슈 해결 아이디어가 있다면 공유 부탁드립니다.

---

## 6. Netlify 리소스 사용 현황 (한도 도달)

잦은 배포 및 디버깅 과정에서 무료 플랜의 월간 크레딧 한도에 도달했습니다. 이로 인해 코드 수정 사항이 실시간으로 배포 서버에 반영되지 않을 수 있습니다.

- **무료 플랜 제공**: 월 300 Credits
- **사용 결과**: **330.2 Credits (한도 초과)**
- **세부 내역**:
    - **Production Deploys**: 총 24회 (성공 22회, 실패 2회) → **330 Credits** (1회당 15 Credits)
    - **Compute / Bandwidth / Web Requests**: 약 0.2 Credits 미만
    - **총 소모량**: 330.2 Credits

> **💡 팀원 주의사항**: 크레딧 한도 초과로 인해 마지막에 푸시한 **'Body 유실 해결 코드'**가 실제 사이트에는 반영되지 않았을 가능성이 높습니다. 신규 기능 확인은 당분간 **로컬 환경**에서 진행해 주시기 바랍니다.
